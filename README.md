# 关于
开发一旦涉及到多人协作，我们通常会制定一些规范（也可叫做基于团队的风格和偏好）不管是口头的还是文档形式的还是公司层面的或者是团队层面的，
面对这些规范或风格往往会收到很多的吐槽，比如：
1.这些规范都是官僚制度下产生的浪费大家的编程时间、影响人们开发效率, 浪费时间的东西。
2.我是个艺术家，手艺人，我有自己的规范和原则。
3.规范不能强求一律，应该允许很多例外。
4.我擅长制定编码规范，你们听我的就好了。
